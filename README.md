# nis-pipo
Аналог when2meet (Проект для НИСа)
Приложение для совместного выбора времени события

Требования к финальному проекту и оценка
1. 4 Балла (За каждый пункт 4/<кол-во пунктов> баллов)

    1. Приложение поддерживает аутентификацию пользователей и контроль доступа к API. +

    2. Приложение имеет gRPC или HTTP API (минимум четыре бизнес-метода: создание, 
    получение, изменение, удаление). +

    3. Все сервисы и API покрыты тестами (unit и функциональными). +

    4. Приложение использует внешнюю БД для хранения пользователей и бизнес-
    информации. +

    5. Схема базы данных создаётся при запуске или деплое приложения, поддерживается
    версионирование схемы. +

    6. Схема базы данных отражается в код при сборке. Несоответствие ORM-моделей,
    запросов и схемы приводит к ошибке сборки. -

2. 4 Балла (За каждый пункт 4/<кол-во пунктов> баллов)

    1. Приложение поддерживает логирование. +

    2. Приложение поддерживает метрики. +

    3. Приложение может быть запущено в Kubernetes (приложение, БД, логирование,
    балансировщик и сервис). -

    4. Поддерживается сборка логов приложения и всех взаимодействующих с приложением
    инфраструктурных объектов в Kubernetes. -

3. 2 Балла (За каждый пункт 2/<кол-во пунктов> баллов)

    1. Каждый коммит в мастер-ветку собирается при помощи CI/CD системы. В случае
    ошибочной сборки об этом сигнализируется каким-либо образом (плашка на GitHub,
    нотификация в чат-бот). + 

    2. По всем API-методам есть Swagger-документация, доступная из приложения. +

# API

# Authentication

POST /api/auth/register — регистрация пользователя

POST /api/auth/login — вход в систему, получение JWT

GET /api/auth/me — получение текущего пользователя

# Meetings

GET /api/meetings — список встреч текущего пользователя

POST /api/meetings — создание встречи

GET /api/meetings/{id} — получение встречи по ID

PUT /api/meetings/{id} — обновление встречи

DELETE /api/meetings/{id} — удаление встречи

GET /api/meetings/{id}/results — получение результатов встречи

PUT /api/meetings/{id}/finalize — финализация встречи


# Participants

POST /api/meetings/{id}/participants — создание участника встречи

PUT /api/meetings/{id}/participants/{participant_id}/slots — сохранение выбранных слотов

# Monitoring

GET /metrics — метрики

Prometheus: http://localhost:9090

GET /swagger/index.html — Swagger



Локальный запуск: make up

Frontend: http://localhost:8081

Swagger: http://localhost:8080/swagger/index.html

Metrics: http://localhost:8080/metrics

Prometheus: http://localhost:9090

Логи: make logs

Остановить проект: make down

Перезапустить проект: make restart

Скачать Go-зависимости: make deps

Собрать backend: make build

Запустить тесты: make test

Сгенерировать Swagger: make swagger

Применить миграции: make migrate-up

Откатить миграции: make migrate-down
